Git & Github 学习指南

发表于 2018-07-09 | 更新于 2018-07-26 | 分类于 教程 | 评论数： | 阅读次数：

本文从入门的角度介绍 Git 和 Github 的基本概念，并给出实例方便上手。
本教程翻译自 LaunchSchool A Guide to Git and Github，仅供个人学习参考。

一. 初识 Git & Github

第一部分将简单介绍什么是 Git 和 Github，它们有什么用，以及我们为什么要用。
首先我们从第一个概念——repo 开始，似乎我们常常可以看到这一词。

什么是 repo?

repo 是 repository 的简称，直译过来名为“存储库”，实际就是存储程序文件的地方。

什么是 Git?

Git 是一个 SCM (Source Control Management) 工具，SCM 又被称为“版本控制系统”（Version Control System）。顾名思义，就是帮助我们在程序开发过程中控制版本的工具，在实际开发过程中十分有用。
Git 与其他 SCM 工具的最大区别在于它是分布式的，这意味着它并没有一个中央代码存储库。正因如此，具有开源特性的 Git 被广泛采用。
你知道吗？Git 是被 Linux 之父 Linus Torvalds 发明的。据说是因为 Linus 觉得现有的版本控制系统不好用，然后顺手写了一个。
Git 是分布式版本控制系统，其特点是不需要服务器/客户端架构及中央存储库。而是所有开发者将项目存在本地，彼此之间直接推送提交。

什么是 Github?

然而实际上大家并不是彼此之间直接提交，而是依赖于一个中央存储库，这个主要存储库（main repo）一般存放在 Github.com 上。尽管有服务器/客户端，但 Git 的核心还是分布式版本控制系统。Github.com 就是一个远程存储库，区别于存储在你计算机内的本地存储库。
对于什么是 Github，直接引用 Github.com：

“GitHub is the best place to share code with friends, co-workers, classmates, and complete strangers. Over six million people use GitHub to build amazing things together.”

简单来说，Github 就是同任何人分享代码的地方。

如何开始

好了，你已经对 Git 和 Github 有了基本的认识。我们接下来将会分别从本地存储库和远程存储库两个部分来学习。
在第一部分，我们将会学习到 Git 的基本操作，包括：创建存储库、追踪变化、分支控制。在第二部分，我们主要学习如何使用 Github，去与他人分享、协作。
那么，安装好 Git，注册一个 Github 账号。把双手放在你的键盘上，我们要开始了！

分支控制

什么是分支？

分支就是代码库中所有文件的拷贝，每一个分支都有其特定的名字、版本集以及提交历史。当你创建一个新的存储库，会自动创建一个默认分支 master。

为什么使用分支？

分支是版本控制工具中的强大机制，能够让开发者在开发过程中游刃有余而无需担心影响主代码库。
举个例子，假设你想对一项新的功能进行实验。但这需要对现有的大量代码进行改动，你不确定新功能是否成功。你想进行尝试，但是有不想影响现有代码。那么你可以创建一个新的分支，然后在它上面进行实验。如果不成功，那么直接删除就行。如果成功，那么可以直接合并到现有的分支如 master 中。

三. 远程存储库

简介

不在我们本地设备上的存储库称为远程存储库。本地存储库可以从远程存储库创建，并且可以与远程存储库推送（push）或提取（pull）提交。我们有很多原因会用到远程存储库，如自己备份，多人协作，甚至部署代码。但最主要的还是为了代码协作，我们将在这一章中介绍如何使用存放在 Github.com 上的远程存储库。

分布式 vs 集中式

在正式介绍 Github.com 之前，先对版本控制系统有个大致认识。因为 Git 的“去中心化”特性，本身并没有提供项目协作的方式。
在一些服务器/客户端 SCM 系统中，开发者总是与服务器打交道，服务器作为整个开发团队的中央存储库。服务器存储项目正式代码，团队成员直接向服务器提交。
在“去中心化” SCM 体统中，如 Git，开发者只需在本地存储库上工作。从理论上讲，分散式SCM系统对单点故障或控制更具免疫力。
然而，在实践中，基于团队的项目需要某种集中控制。 这就是 Github.com 的用武之地。大多数团队使用代码库的本地副本，但使用Github.com托管的存储库作为“集中式”存储库，它作为整个代码库的唯一真实来源。

多种选择

注意不一定非得 Github.com。开发团队可以使用其他服务或应用，甚至本地 Git 存储库作为中央存储库。
但你开始寻找存储库托管服务，你将发现有许多选择，比如：

  • Gitlab
  • Bitbucket
  • Kiln
  • Codeplane
  • GitHub

我们主要介绍 Github。

  1. 团队中多人推送代码到远程存储库。
  2. 你在另外一台机器推送到远程存储库。
  3. 你的项目在 Github 上公开，其他人有贡献。
