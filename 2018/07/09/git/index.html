Git & Github 学习指南

发表于 2018-07-09 | 分类于 教程

本教程翻译自 LaunchSchool A Guide to Git and Github，仅供个人学习参考。
欢迎转载，请注明出处。

一. 初识 Git & Github

第一部分将简单介绍什么是 Git 和 Github，它们有什么用，以及我们为什么要用。
首先我们从第一个概念——repo 开始，似乎我们常常可以看到这一词。

什么是 repo?

repo 是 repository 的简称，直译过来名为“存储库”，实际就是存储程序文件的地方。

什么是 Git?

Git 是一个 SCM (Source Control Management) 工具，SCM 又被称为“版本控制系统”（Version Control System）。顾名思义，就是帮助我们在程序开发过程中控制版本的工具，在实际开发过程中十分有用。
Git 与其他 SCM 工具的最大区别在于它是分布式的，这意味着它并没有一个中央代码存储库。正因如此，具有开源特性的 Git 被广泛采用。

你知道吗？Git 是被 Linux 之父 Linus Torvalds 发明的。据说是因为 Linus 觉得现有的版本控制系统不好用，然后顺手写了一个。
Git 是分布式版本控制系统，其特点是不需要服务器/客户端架构及中央存储库。而是所有开发者将项目存在本地，彼此之间直接推送提交。

什么是 Github?

然而实际上大家并不是彼此之间直接提交，而是依赖于一个中央存储库，这个主要存储库（main repo）一般存放在 Github.com 上。尽管有服务器/客户端，但 Git 的核心还是分布式版本控制系统。Github.com 就是一个远程存储库，区别于存储在你计算机内的本地存储库。
对于什么是 Github，直接引用 Github.com：

“GitHub is the best place to share code with friends, co-workers, classmates, and complete strangers. Over six million people use GitHub to build amazing things together.”

简单来说，Github 就是同任何人分享代码的地方。

如何开始

好了，你已经对 Git 和 Github 有了基本的认识。我们接下来将会分别从本地存储库和远程存储库两个部分来学习。
在第一部分，我们将会学习到 Git 的基本操作，包括：创建存储库、追踪变化、分支控制。在第二部分，我们主要学习如何使用 Github，去与他人分享、协作。
那么，安装好 Git，注册一个 Github 账号。把双手放在你的键盘上，我们要开始了！

密码

当你在本地库做出改变，想要推送到远程库。如果每次都需要输入密码，将会显得很繁琐。Github 推荐了两种方法来处理验证，我们将简单介绍。

密码缓存

你应该记得我们在 git remote 命令中添加远程库 URL时，我们用了 http://。密码缓存只能使用 http:// URL。Github 提供了详细如何设置。

SSH

你也可以选择使用 SSH 连接 Github 存储库。Github 也提供了如何设置并生成SSH keys。

四. 结论

本教程的目的是让你能够快速上手并且对核心概念有基本理解。还有许多内容这里没有涉及。我们希望你能够回顾一下全部内容，并能熟练使用 Git。高级的操作如分支、合并、冲突等对有经验的开发者也有些困难，你将在以后慢慢接触到这些话题。
